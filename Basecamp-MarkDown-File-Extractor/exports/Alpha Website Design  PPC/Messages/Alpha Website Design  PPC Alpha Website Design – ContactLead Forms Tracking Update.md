**Deliverables for Alpha Website Design – Contact/Lead Forms Tracking Update**

**1. Updated Forms**

* All existing contact/lead forms on the Alpha website have been updated to include **six additional hidden tracking fields**:

  + utm\_source
  + utm\_medium
  + utm\_campaign
  + utm\_content
  + utm\_term
  + gclid
* These fields are **hidden from users** but will be included in the form submission payload.

**2. Front-End Tracking Script**

* A JavaScript snippet has been implemented on all pages where forms exist.
* The script performs the following:

  1. **Reads UTM parameters and gclid** from the URL.
  2. **Persists values** in cookies/localStorage to maintain tracking across pages.
  3. **Injects values** into the hidden form fields automatically when a user submits the form.

**3. Implementation Note / Client Update**

* **Forms Updated:** All contact/lead forms on the Alpha website (e.g., Contact Us page, Lead Capture forms in the header/footer).
* **Captured Fields:** utm\_source, utm\_medium, utm\_campaign, utm\_content, utm\_term, gclid.
* **How Values Are Populated:**

  + On page load, the script checks URL parameters.
  + If present, values are saved in localStorage/cookies.
  + On form submission, hidden fields are automatically populated with these stored values.
* **Viewing Captured Data:**

+ All submitted values are included in the standard form submission payload.
+ Clients can view these values in their backend form submission dashboard, CRM, or email notifications, depending on form integration.
